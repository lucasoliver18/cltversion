Gerenciamento de Configuração de Software


Controle de Versão

O Controle de Versão nada mais é que o método usado para fazer alterações e atualizões no código fonte de um software. Está prática é muito utilizada por trazer segurança e integridade para o código principal, pois as alterações são feitas em um código secundário que após passar pelos testes de softwares é adicionado ao código principal todas as mudanças feitas, assim protegendo o código de problemas desnecessários.

Controle de Mudanças

O processo de controle de mudanças tem como intuito lidar com solicitações de alteração em projetos e grandes iniciativas. Este processo é de extrema importância, pois atualmente vivemos em mundo de constantes mudanças e muitas delas podem impactar certos projetos. Por conta disto, o objetivo do Controle de Mudanças é tentar minimizar o impacto das mudanças no processos, fazendo com que as mudanças sejam recebidas de uma forma natural e que o impactos que vem com elas sejam contidos ao máximo.

Integração Contínua

A integração contínua é uma prática de desenvolvimento de software em que os desenvolvedores, juntam suas alterações de código em um repositório central. Está prática se conecta com o Controle de Versão, pois a equipe de desenvolvedores dividem entre si o'que cada um será responsável de fazer, cada desenvolvedor trabalha em uma parte do código, sem ser o código principal para evitar transtornos desnecessários e após todos terem finalizado as alterações, eles juntam a parte de cada um em um único código para ser implementado na main.
